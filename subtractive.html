Description:

The component generates Sine Waves and places them into a wavetable. From the wavetable, the frames are edited and altered to achieve any desired and required effects. The Subtract Wave class generates the sine waves and places them into the wavetable. The Subtractive Instrument can play more than 1 note at a time (Polyphony). The Subtract Wave Class also takes in any information from the xml score files and alters the wavetable with that information. The type of wave produced by Subtract Wave Class depends upon the info passed by score file. The Subtract Wave class also contains the Reson Filter which can have parameters altered by the score file. The Subtract Instrument class pulls in all the information from the score file and creates a Subtract Wave Object to create and edit sound. The Instrument can play 3 different types of waves: Triangle, Square, and Sawtooth. The sound frames are placed into a ADSR envelope in the Subtractive Instrument class (function called ADSR) which will prevent clicking and popping with its functionality. Moog sound is implemented by altering freq input into sin waves.

Attributes for the score file:

wavetype: sets wavetype
reson: reson freq
bandwidth: reson bandwidth
sweep: moog sound
vibfreq: vibrato freq input
vibamp: vibrato freq amp

Grading criteria:

10 - Waveform playback from tables - Done. In Subtract Wave Class (createwavetable()).
20 - Varying pitch playback from tables - Done. In Subtract Wave Class (createwavetable()) - parameters passed in score file.
30 - Envelope generation - Done. In Subtractive Instrument Class - each frame is passed to ADSR to be edited.
35 - Polyphony - Done. More than one note can be played at a time.
40 - Reson implementation and the Moog sound - Done. Reson Filter in Subtract Wave class - parameters passed in score file. Moog Sound done by using sweep parameter in score and set in createwavetable().
50 - Filter envelopes - Done.
